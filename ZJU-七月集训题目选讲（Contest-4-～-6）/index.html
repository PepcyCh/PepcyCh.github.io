Contest 4 - B - Positive Eigenvalues

题目大意

给出一个 n×nn \times nn×n 的 010101 矩阵，判断其是否为正定矩阵。

1≤n≤501 \leq n \leq 501≤n≤50

1≤T≤1,0001 \leq T \leq 1,0001≤T≤1,000，n≥10n \geq 10n≥10 的子测试点不超过 101010 个。

时间限制：1s1 \text{s}1s，内存限制：64MB64 \text{MB}64MB。

题解

\begin{align} &\because \lambda_i > 0 ~ (1 \leq i \leq n) \\ &\therefore \det(\textbf{A}) = \prod_{i = 1}^{n} \lambda_i > 0 \\ &\because \prod_{i = 1}^{n} \lambda_i \leq (\frac{\sum_{i = 1}^{n} \lambda_i}{n})^n = (\frac{\sum_{i = 1}^{n} \textbf{A}_{i, i}}{n})^n \leq 1 ~ (\textbf{A}_{i, i} \text { is either } 1 \text{ or } 0) \\ &\therefore \det(\textbf{A}) = 1 \\ &\therefore \lambda_i = 1 ~ (1 \leq i \leq n), \textbf{A}_{i, i} = 1 ~ (1 \leq i \leq n) \end{align}

根据行列式计算的定义式，我们知道，矩阵 A\textbf{A}A 的除主对角线外其他 1∼n1 \sim n1∼n 的置换上的位置的值不全为 111。即，以该矩阵为某一有向图的邻接矩阵，该有向图无环（自环除外）。判断主对角线上是否全为 111 ，然后建图跑拓扑排序判断是否存在环即可。

Contest 4 - C - String Set

题目大意

给出一个字符串的可重集合 AAA（初始为空），定义一个字符串的最长匹配前缀长度（MMPL）为：

MMPL(S)=max⁡(∣T∣,T∈A and T is a prefix of S)\text{MMPL}(S) = \max(|T|, T \in A \text{ and } T \text{ is a prefix of } S) MMPL(S)=max(∣T∣,T∈A and T is a prefix of S)

有 nnn 次操作，每次操作向 AAA 中放入一个字符串 sss，或删除一个已有的字符串。有 mmm 次询问。每次询问一个字符串 SSS 在操作区间 [l,r][l, r][l,r] 内 MMPL(S)\text{MMPL}(S)MMPL(S) 变化了几次（操作从头开始，但只在询问区间计数）。

所有字符串为 010101 串。

1≤n,m≤1051 \leq n, m \leq 10^51≤n,m≤105

1≤∣s∣≤32,∣S∣=321 \leq |s| \leq 32, |S| = 321≤∣s∣≤32,∣S∣=32

1≤T≤101 \leq T \leq 101≤T≤10，大测试点的数目不会超过 444。

时间限制：2s2 \text{s}2s，内存限制：1024MB1024 \text{MB}1024MB。

题解

离线所有的操作与查询（拆为两个）并按时间排序。建立一个 Trie。

插入时直接插入，给其单词计数器值加 111。如果是第一次插入该字符串，打一个 −1-1−1 的 tag，给其贡献计数器值加 111。

删除入沿 Trie 走到末尾节点，给其单词计数器值减 111。如果此时计数器为 000，打一个 −1-1−1 的tag，给其贡献计数器加 111。

查询时沿 Trie 走，累加贡献计数器的值即可。

传递标记时：如果其无子节点，不传递；如果子节点的单词计数器为 000，将 tag 的值加给子节点的 tag；如果子节点的单词计数器不为 000，将 tag 的值加给子节点的贡献计数器。

Contest 4 - E - Candies

题目大意

Alice 和 Bob 分 nnn 支蜡烛，每只蜡烛有一个价值 wiw_iwi​。两人分得的蜡烛数目应尽量相等，在此基础上，两人分得的蜡烛的价值和应尽量相等。若无法让二人的蜡烛价值和相等，Alice 分得的蜡烛价值和应小于 Bob 的。求二人分得的蜡烛价值和。

1≤n≤1001 \leq n \leq 1001≤n≤100

∑i=1nwi≤100,000\sum_{i = 1}^{n} w_i \leq 100,000∑i=1n​wi​≤100,000

1≤T≤1001 \leq T \leq 1001≤T≤100

时间限制：2s2 \text{s}2s，内存限制：64MB64 \text{MB}64MB。

题解

f(i,j)f(i, j)f(i,j) 表示数 iii 是否可有 jjj 个数拼成，第二维可用一个 long long 或 std::bitset 储存，则转移为： f[i] |= (f[i - w[i]] << 1)。

计算答案时从 ∑wi/2\sum w_i / 2∑wi​/2 开始向下枚举，判断是否能由一半的 nnn 的个数（nnn 为奇数时要判断两个）的数拼成即可。

Contest 5 - B - MUJI

题目大意

给定一棵 nnn 个点的树，每个点上有一个权值 viv_ivi​。有 qqq 次询问，每次询问两个点，从一个点开始，沿它们间的简单路径走到另一个点，每到一个点（含端点），就可以选择获得或失去 kvikv_ikvi​ 个物品（k∈Nk \in \mathbb{N}k∈N，自行选择），求最终所持有物品的正最小值。

1≤n,q≤200,0001 \leq n, q \leq 200,0001≤n,q≤200,000

1≤T≤101 \leq T \leq 101≤T≤10，大测试点不超过 333 个。

时间限制：2s2 \text{s}2s，内存限制：128MB128 \text{MB}128MB。

题解

由裴蜀定理，答案即路径上顶点权值的最大公约数，树上倍增即可。

Contest 5 - C - Magical Tree

题目大意

给定一棵 nnn 个点的树，每个点是黑色或白色，每条边有一个权值 cic_ici​。要求删去权值和最小的边集使得每一个连痛快内最多有一个黑点。同时有 mmm 次操作，每次操作会把一个点变为黑点（该点原来可能为黑点）。求操作前以及每次操作后，需要删去的边集的最小权值和。

1≤n,m≤100,0001 \leq n, m \leq 100,0001≤n,m≤100,000

1≤ci≤1,0001 \leq c_i \leq 1,0001≤ci​≤1,000

1≤T≤101 \leq T \leq 101≤T≤10

时间限制：2s2 \text{s}2s，空间限制：64MB64 \text{MB}64MB。

题解

考虑把操作倒过来，即把黑点变为白点，则每次需要恢复一条与该点所在连通块相连的、权值最小的边。一开始让所有的点为黑点，即所有的边都删去，为每一个连通块维护一个可并堆（小根堆），在恢复边时，选择堆顶的边，并弹出、标记这条边，将被合并的两个连通块的可并堆合并。连通块可用并查集维护。注意处理改变前后颜色不变的情况。

Contest 5 - F - Simple Counting

题目大意

求有多少个 XXX 满足：

⌊XK⌋  ∣  X (X∈[1,N],X∈Z)\lfloor \sqrt[K]{X} \rfloor \;|\; X ~ (X \in [1, N], X \in \mathbb{Z}) ⌊KX​⌋∣X (X∈[1,N],X∈Z)

其中 a∣ba | ba∣b 表示 bbb 是 aaa 的倍数。

1≤N≤231−11 \leq N \leq 2^{31} - 11≤N≤231−1

2≤K≤42 \leq K \leq 42≤K≤4

1≤T≤1,000,0001 \leq T \leq 1,000,0001≤T≤1,000,000

时间限制：2s2 \text{s}2s，空间限制：64MB64 \text{MB}64MB。

题解

[xK,(x+1)K)[x^K, (x + 1)^K)[xK,(x+1)K) 内的数的 KKK 次根号的值均为 xxx，该区间内合法的数的个数为：

(x+1)K−xK−1x+1\frac{(x + 1)^K - x^K - 1}{x} + 1 x(x+1)K−xK−1​+1

按 KKK 分类手推式子，对于最后一个不完整的区间单独考虑即可。

Contest 6 - A - SmartLy’s Game

题目大意

两个人玩游戏：有 nnn 堆石子，每堆有一些石子。先手可以在石子数不少于 AAA 的石堆中取走 AAA 个石子，后手可以在石子数不少于 BBB 的石堆中取走 BBB 个石子。一方不能行动则另一方获胜，问先手是否必胜。

1≤N≤1001 \leq N \leq 1001≤N≤100

1≤A,B≤1091 \leq A, B \leq 10^91≤A,B≤109

1≤T≤10,0001 \leq T \leq 10,0001≤T≤10,000

时间限制：1s1 \text{s}1s，空间限制：64MB64 \text{MB}64MB。

题解

每堆石子数可以对 A+BA + BA+B 取模而对答案无影响。取模后按石子数分类：

  • ai<min⁡(A,B)a_i < \min(A, B)ai​<min(A,B)：双方都不能取，可以无视；
  • min⁡(A,B)≤ai<max⁡(A,B)\min(A, B) \leq a_i < \max(A, B)min(A,B)≤ai​<max(A,B)：只有小的一方可以取，如果存在这样的堆，则小的一方必胜；
  • ai≥max⁡(A,B) and ai≥2min⁡(A,B)a_i \geq \max(A, B) \text{ and } a_i \geq 2\min(A, B)ai​≥max(A,B) and ai​≥2min(A,B)：大的一方只能取一次，而小的一方可以取多次。当小的一方取过一次后转化为情况二，而大的一方取过一次后转化为情况一。如果存在这样的堆且先手为小者，先手必胜；存在至少 222 个这样的堆且后手为小者，后手必胜；
  • ai≥max⁡(A,B)a_i \geq \max(A, B)ai​≥max(A,B)：当不能根据之前的情况判定胜负时，每一个这种堆只能被双方取一次（只存在一个情况三的堆且后手为小者时，先手会先抢走这堆），故根据这种堆数目的奇偶判断胜负。

Contest 6 - E - Heltion and Sequence

题目大意

给定一个初始有 nnn 个元素的数组 {ai}\{a_i\}{ai​}，之后有 qqq 个操作。一共有 444 种操作：

  • 1 u v：在位置 uuu 前插入数 vvv；
  • 2 l r：询问区间 [l,r][l, r][l,r] 的元素和；
  • 3 u v：将位置 uuu 的数改为 vvv；
  • 4：按升序排序数组。

1≤n,q≤200,0001 \leq n, q \leq 200,0001≤n,q≤200,000

0≤v,ai≤327670 \leq v, a_i \leq 327670≤v,ai​≤32767

∑n≤233,333, ∑q≤466,666\sum n \leq 233,333, ~ \sum q \leq 466,666∑n≤233,333, ∑q≤466,666

时间限制：10s10 \text{s}10s，空间限制：1024MB1024 \text{MB}1024MB。

题解

除操作四外均为 Splay 的基本操作。我们每修改或插入一个数后，将对应节点放入一个修改数组，排序时将修改数组内的点全部删除，然后重新插入到 Splay 里即可。

Contest 6 - G - Heltion and Backpack

题目大意

有 nnn 种物品，每种物品的价值是 viv_ivi​，每种物品有无数个，求取出正好 kkk 个物品能得到多少种价值和。

1≤n,k,vi≤1,0001 \leq n, k, v_i \leq 1,0001≤n,k,vi​≤1,000

1≤T≤10, ∑max⁡(n,k,vi)≤3,3331 \leq T \leq 10, ~ \sum \max(n, k, v_i) \leq 3,3331≤T≤10, ∑max(n,k,vi​)≤3,333

时间限制：3s3 \text{s}3s，空间限制：512MB512 \text{MB}512MB。

题解

对于每个价值为 viv_ivi​ 的物品，令多项式 f(x)f(x)f(x) 中 xvix^{v_i}xvi​ 的系数为 111，则答案为 f(x)kf(x)^kf(x)k 的非零系数个数。
