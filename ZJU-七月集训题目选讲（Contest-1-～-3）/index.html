Contest 1 - A - Saber

题目大意

给定一棵 nnn 个节点的树，每条边有一个权值 www。有 QQQ 次询问，每次询问两个点，判断它们之间的路径上的边中，是否存在三条边使得它们的权值可以组成一个三角形。

1≤n,Q≤100,0001 \leq n, Q \leq 100,0001≤n,Q≤100,000

1≤w≤1091 \leq w \leq 10^91≤w≤109

1≤T≤51 \leq T \leq 51≤T≤5

时间限制：2s2 \text{s}2s，内存限制：64MB64 \text{MB}64MB。

题解

权值最小的一段不能组成三角形的权值是斐波那契数列，而斐波那契数列的第 454545 项已经超过了 10910^9109 ，所以，当路径上的边数超过 454545 时即可直接输出 YES，否则暴力判断。

Contest 1 - B - Lancer

题目大意

有 nnn 个物品，每个物品有一个字符（小写字母）和一个权值（1∼91 \sim 91∼9）。从这些物品中连续取出 i×li \times li×l 个，其中 i≥Si \geq Si≥S，l∈[L,R]l \in [L, R]l∈[L,R]，并且满足对于这 iii 个长为 lll 的子段，不存在某两个子段在某一个相对位置上的字符相同，可获得它们权值和的价值。求可获得的最大价值。

1≤n≤500,0001 \leq n \leq 500,0001≤n≤500,000

0≤R−L≤100 \leq R - L \leq 100≤R−L≤10

1≤T≤10,∑n≤2,500,0001 \leq T \leq 10, \sum n \leq 2,500,0001≤T≤10,∑n≤2,500,000

时间限制：2s2 \text{s}2s，内存限制：64MB64 \text{MB}64MB。

题解

注意到 R−LR - LR−L 很小，我们可以枚举长度 lll。对于每一个长度，暴力搞出从某一个位置开始，每次向后跳 lll 个物品所能到的最远距离，答案就是每一个长为 lll 的连续子段中该最远距离的最小值，可以用单调队列来维护。

Contest 1 - G - Berserker

题目大意

求从 1∼n1 \sim n1∼n 中选出不超过 kkk 个不同的数满足它们的乘积不是除 111 外其他完全平方数的倍数的方案数，答案对 1,000,000,0071,000,000,0071,000,000,007 取模。

1≤n,k≤5001 \leq n, k \leq 5001≤n,k≤500

1≤T≤101 \leq T \leq 101≤T≤10

时间限制：2s2 \text{s}2s，内存限制：512MB512 \text{MB}512MB。

题解

对每个数进行质因数分解并按质因子分组，每组最多选一个数以保证乘积无平方因子（忽略有平方因子的数）。但有一些数可以被分到多个组中。

注意到 192=361,232=52919^2 = 361, 23^2 = 529192=361,232=529，即只有 {2,3,5,7,11,13,17,19}\{2, 3, 5, 7, 11, 13, 17, 19\}{2,3,5,7,11,13,17,19} 的倍数会被分到多组。除去这些质因子后，每个数可以被分到唯一的组。

对于这 888 个质因子，用二进制位的方式保证乘积无平方因子。

具体地说，对于只含这 888 个质因子的数，我们进行如下 DP 转移：

f(j,s)=∑gi⊂sf(j−1,s⊕gi)f(j, s) = \sum_{g_i \subset s} f(j - 1, s \oplus g_i) f(j,s)=gi​⊂s∑​f(j−1,s⊕gi​)

其中 f(i,s)f(i, s)f(i,s) 表示选择了 iii 个数，它们乘积的质因子为 sss（状压表示）的方案数，gig_igi​ 表示数 iii 的质因子（状压表示）。

对于其他的组，也进行类似的 DP，但是其他组需要满足组内只能选一个数。我们可以通过改变枚举顺序来实现。

Contest 2 - C - Ibrahimovic and the Angry Cashier

题目大意

有 mmm 张一元的硬币和无限张一百元的纸币，进行 nnn 次购物，每次的价格为 cic_ici​，每次售货员会获得 xi×wix_i \times w_ixi​×wi​ 的怒气值，其中 xix_ixi​ 为本次需要找零的钱数。售货员找零时只会给一百元的纸币和少于 100100100 个的一元的硬币，并且这些钱可以用于后续的购物。求 nnn 次购物后售货员怒气值的最小值。

1≤n,ci,wi≤100,0001 \leq n, c_i, w_i \leq 100,0001≤n,ci​,wi​≤100,000

0≤m≤1090 \leq m \leq 10^90≤m≤109

1≤T≤201 \leq T \leq 201≤T≤20

时间限制：1s1 \text{s}1s，内存限制：256MB256 \text{MB}256MB。

题解

显然，所有的 cic_ici​ 可以对 100100100 取模。每次都用硬币付款，当硬币不够时，可以从之前的几次中选择未选择过的、 (100−ci)wi(100 - c_i)w_i(100−ci​)wi​ 最小的一次，使用纸币付款，相当于换到了 100100100 个硬币。用优先队列维护最小的 (100−ci)wi(100 - c_i)w_i(100−ci​)wi​ 即可。

Contest 3 - A - include <set>

题目大意

给定 nnn 个点，每个点有一个质量 mim_imi​。你可以平移或旋转点集，但必须：

  • 保持各个点之间的相对位置不变；
  • 点的纵坐标不得为负。

以 xxx轴为零势面，最小化点的重力势能和。

0≤n,x,y≤10,0000 \leq n, x, y \leq 10,0000≤n,x,y≤10,000

0≤m≤1000 \leq m \leq 1000≤m≤100

时间限制：1s1 \text{s}1s，内存限制：256MB256 \text{MB}256MB。

题解

答案最小时，凸包的一条边与 xxx轴重合。求出重心和凸包，然后求出重心到凸包上的边的最短距离即可。

Contest 3 - B - include <numeric>

题目大意

求 [L,R][L, R][L,R] 内有多少数满足偶数位和与奇数位和的最大公约数不超过 kkk。

1≤L≤R≤10181 \leq L \leq R \leq 10^{18}1≤L≤R≤1018

1≤T≤2,0001 \leq T \leq 2,0001≤T≤2,000

时间限制：1s1 \text{s}1s，内存限制：256MB256 \text{MB}256MB。

题解

直接数位 DP 会超时。

因为 kkk 相同的点，数为 DP 的数组在不顶头时的值是相同的，所以我们先读入所有的测试点，按 kkk 排序，同一 kkk 下 DP 时只清空顶头的位置的值即可。

（「顶头」指下一位的数不能随便选的位置）

1≤m≤40,0001 \leq m \leq 40,0001≤m≤40,000

1≤wi≤10,0001 \leq w_i \leq 10,0001≤wi​≤10,000

1≤T≤51 \leq T \leq 51≤T≤5

时间限制：1s1 \text{s}1s，内存限制：256MB256 \text{MB}256MB。

题解

给与 111 号顶点相邻的顶点重新编号，按二进制位分类设为起点与终点跑多源最短路并更新答案。

Contest 3 - E - #include <deque>

题目大意

要清空一个目前有 nnn 个元素的双端队列，每个元素为 wiw_iwi​。从左侧弹出元素会有 wi×Lw_i \times Lwi​×L 的不高兴值，从右侧则会有 wi×Rw_i \times Rwi​×R 的不高兴值。如果本次与上次均从左侧弹出，会有 qLq_LqL​ 的额外不高兴值，右侧则会有 qRq_RqR​ 的额外不高兴值。求最小的不高兴值。

1≤n≤100,0001 \leq n \leq 100,0001≤n≤100,000

1≤L,R,qL,qR≤1001 \leq L, R, q_L, q_R \leq 1001≤L,R,qL​,qR​≤100

时间限制：2s2 \text{s}2s，内存限制：64MB64 \text{MB}64MB。

题解

以双端队列的某个位置位为界，左侧的只能从左侧弹出，右侧的只能从右侧弹出，为了最小化不高兴值，弹出方式只能为 LRLRLRLLLLLRLRLRLLLLLRLRLRLLLL 或 RLRLRLRRRRRLRLRLRRRRRLRLRLRRRR 。求出前缀后缀和，枚举分界线计算即可。

Contest 3 - G - #include <bits/stdc++.h>

题目大意

做一个支持区间合并的并查集，有 nnn 个元素， qqq 次操作，操作有四种：

  • 合并元素 xxx 与元素 yyy
  • 合并 [x,y][x, y][x,y] 内的元素
  • 询问 xxx 与 yyy 是否合并
  • 询问 xxx 所在的块有多少个元素

1≤n,q≤100,0001 \leq n, q \leq 100,0001≤n,q≤100,000

1≤T≤51 \leq T \leq 51≤T≤5

时间限制：2s2 \text{s}2s，内存限制：64MB64 \text{MB}64MB。

题解

用一个线段树维护并查集的 fa 数组。合并两个元素时修改单点；合并区间时，递归到区间内的元素的 fa 值相等时才进行修改。

由于一段区间之可能从不等变为相等，不可能从相等变为不等，所以区间修改时并不需要懒标记。
